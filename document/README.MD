## Общее описание сервиса

### Цель создания сервиса

Обеспечить работу с нормативами и справочниками.

### Функции

Справочники:
* редактирование
* просмотр/поиск
* история изменений

Нормативы
* редактирование значения номатива
* просмотр/поиск
* история изменений

## Архитектура

Стандартное слоеное приложение.
API вынесено в отдельный модуль для создания библиотеки для внешних потребителей.
Справочники хранятся в виде одной таблицы.
Нормативы представляют собой иерахический справочник. 
При этом для определения следующего уровня требуется указать все предыдущие.

## Конфигурация

Специфичные параметры:

Структуру см в [application.yml](src/main/resources/application.yaml)

|Название параметра                              |Описание                                                                                                                 |
|:-----------------------------------------------|:------------------------------------------------------------------------------------------------------------------------|
|spring.datasource.proxy-auth                    |Флаг для включения proxy authentification к Oracle (требует дополнительной настройки пользователей в БД)                 |
|jwt.secret                                      |Секрет в jwt                                                                                                             |
|jwt.ttl                                         |Время жизни токена в милисекундах                                                                                        |
|jwt.subject                                     |Subject в jwt                                                                                                            |
|jwt.keys                                        |Ключи, под которыми хранятся соответствующие данные в jwt токене                                                          |


## Зависимости

Сервис требует доступ к БД (параметры подключения стандартные для spring boot стека)
